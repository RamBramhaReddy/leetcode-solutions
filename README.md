## Setup: Auto-Update System

This repository uses **GitHub Actions** + **Python script** to automatically update the README whenever you push new solutions.

### How It Works

1. **Push a new solution folder** (e.g., `2345-my-problem/`)
2. **GitHub Actions** automatically triggers the Python script
3. **Script scans** all problem folders and extracts problem numbers & titles
4. **README.md** updates automatically with the new table
5. **Changes are committed** and pushed back to main branch

### Files Needed

#### 1. Python Generator Script  
**Location**: `.github/scripts/generate_readme.py`

#### 2. GitHub Actions Workflow  
**Location**: `.github/workflows/update-readme.yml`

## Quick Start

### Step 1: Create Directories
```bash
mkdir -p .github/scripts
mkdir -p .github/workflows
```

### Step 2: Add Python Script
Create `.github/scripts/generate_readme.py` (see below)

### Step 3: Add GitHub Actions Workflow
Create `.github/workflows/update-readme.yml` (see below)

### Step 4: Commit and Push
```bash
git add .github/
git commit -m "setup: add auto-readme generation workflow"
git push origin main
```

### Step 5: Add New Problems
Just push a new problem folder like `9999-my-problem/` with a `solution.py` file, and the README will update automatically!
